Introduction

This page demonstrates how to create a clean and well-structured web layout using only semantic HTML5 elements. It's designed with accessibility and search engines in mind, making it readable by both humans and machines.

Core Principles

1. Semantic Structure

Use tags like <header>, <nav>, <main>, and <footer> to give your content meaning. Avoid generic containers unless necessary.

2. Accessibility

Include ARIA labels, use landmarks, and follow proper heading structure. Provide meaningful text for screen readers, and ensure all content is keyboard-navigable.

3. SEO Best Practices

Write a descriptive title and meta description, use one <h1> tag per page, and organize subtopics with <h2> and <h3> as needed.
